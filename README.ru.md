# No Pain, Same Gain — IGA

[English](README.md) | [Português (BR)](README.pt-BR.md) | **Русский** | [中文](README.zh-CN.md)

**Interference-Gated Aggregation (IGA)** — рекуррентная модель с растущей памятью, рекуррентное состояние которой постоянно по длине последовательности, без затрат perplexity и с recall, равным полному вниманию на самой сложной стандартной задаче из литературы. Основывается на и улучшает фундаментальную формулировку растущей памяти GRM / Memory Caching (Behrouz et al., 2026).

**Автор:** Фабио де Оливейра Перес — независимый исследователь, Рио-Верди, Гояс, Бразилия.

## Суть одним предложением
RNN с растущей памятью [Behrouz et al., 2026] возвращают рекуррентность, но разрушаются на multi-query associative recall (MQAR), а их состояние растёт без границы. IGA устраняет разрушение (гейт *ненормализованного* посегментного состояния внутри одной глобальной дроби, с фиксированной памятью как доказуемым нижним порогом инициализации), маршрутизирует по *выученной идентичности* через полиномиальный маршрутизатор на тензорном скетче (без чтения token IDs) и делает состояние **постоянным по T** (факторизация ранга ≤ S точно устраняет член d²; ограничение памяти ограничивает рост) — 0,39 МБ при d=64, в 15× меньше KV-кэша, спроецированного на 1 трлн параметров, без потери recall на ключевой задаче. На официальном Zoology MQAR-стенде, самая сложная стандартная настройка (512,64): IGA 99,2 против Mamba 23,2 и Based 79,5, на уровне полного softmax-внимания 100,0. На реальной символьной языковой модели (TinyStories, 3 seed, два контекста) IGA сравнима с базовой моделью фиксированной памяти той же ширины по bits-per-character в пределах шума между seed — **постоянная память без затрат perplexity**.

## Что означает название
Название точно отражает то, что подтверждают измерения: *no pain* — растущая память и шаг O(T²) устранены, остаётся постоянное состояние и O(T) вычислений на шаг; *same gain* — recall равен полному вниманию на самой сложной стандартной задаче, а perplexity равен базовой модели фиксированной памяти на реальном тексте.

## Честные ограничения (§8 статьи)
Преимущество ассоциативного recall зависит от задачи (не переносится на planted-fact зонд на естественном тексте при d=128); предварительный прирост ~0,016 bpc, упоминавшийся в другом месте, *не* воспроизводится в протоколе 3 seed, 4000 шагов; масштабирование d/ctx для проявления различия recall на реальном тексте оставлено на будущую работу (ограничено бюджетом GPU). Утверждения о recall получены в малом масштабе (один слой, d=64); эксперимент с языковой моделью — единственная настройка на реальном тексте.
